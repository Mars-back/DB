# База даних ресторану

Проєктування бази даних для системи управління рестораном, де відстежуються клієнти, столики, позиції меню, а також обробляються замовлення та детальні чеки.

---

## Схема бази даних

### Таблиці

| Таблиця | Опис |
|---|---|
| `Clients` | Клієнти - ім'я, телефон, email |
| `Tables` | Столики - номер столика, місткість, розташування |
| `Menu_Items` | Позиції меню - назва, опис, ціна, категорія |
| `Orders` | Замовлення - зв'язок клієнта зі столиком, час замовлення, статус |
| `Order_Items` | Позиції замовлення - які страви і в якій кількості є в кожному замовленні |

## OLTP запити

### INSERT
- Додавання нового клієнта
- Додавання нової позиції в меню

### UPDATE
- Оновлення номеру телефону клієнта
- Зміна статусу замовлення на «Завершено»

### DELETE
- Видалення конкретної позиції з певного замовлення
- Видалення запису про клієнта

### SELECT
- Підготувати чек для замовлення (перелік страв, кількість, ціна, сума)
- Знайти всі не зайняті столики (столики, для яких немає незавершених замовлень)

---

## OLAP запити

| Запит | Опис |
|---|---|
| Загальний дохід | Обчислити загальний денний дохід за датами за останній місяць |
| Популярні страви | Знайти топ-10 найбільш популярних позицій меню |
| Вартість за часом | Обчислити середню вартість замовлення за часом доби (сніданок, обід, вечеря) |
| Найбільші витрати | Визначити клієнтів з найбільшими загальними витратами (через CTE) |
